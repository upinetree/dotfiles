# Meeting Time

今日のGoogleカレンダーを取得し、ミーティング一覧・合計時間・集中できる空き時間を表示する。

## 手順

### 1. 今日の予定を取得

`mcp__claude_ai_Google_Calendar__list_events` を呼び出す：
- `startTime`: 今日の `YYYY-MM-DDT00:00:00`（JST）
- `endTime`: 今日の `YYYY-MM-DDT23:59:59`（JST）
- `timeZone`: `Asia/Tokyo`
- `orderBy`: `startTime`

### 2. イベントをフィルタリング・分類

以下を除外する：
- 終日イベント（`dateTime` フィールドがなく `date` フィールドのみのもの）
- ステータスが `cancelled` のイベント
- 自分が `declined` したイベント（attendees 内の self:true のエントリの responseStatus が `declined`）

残ったイベントをさらに分類する：
- **確定ミーティング**: self の responseStatus が `accepted`、または自分が organizer（attendees なし含む）
- **未回答ミーティング**: self の responseStatus が `needsAction` または `tentative`

### 3. 出力を構成

## 出力形式

以下のフォーマットで出力する：

```
## 今日のミーティング（M月D日）

| 時間 | イベント |
|------|---------|
| HH:MM〜HH:MM | イベント名 |
| HH:MM〜HH:MM | イベント名 |

**合計: X時間Y分**（または **合計: Y分** ）

### 未回答（要確認）
| 時間 | イベント |
|------|---------|
| HH:MM〜HH:MM | イベント名 |

※ 未回答のイベントがない場合はこのセクションを省略する

## 集中できる時間帯
- HH:MM〜HH:MM（X時間Y分）
- HH:MM〜HH:MM（Y分）

ミーティングがない場合は「今日はミーティングがありません」と表示する。
```

## 合計時間の計算ルール

- **確定ミーティングのみ**を合計に含める（未回答イベントは含めない）
- 時間帯が重なるイベントは時間帯をマージし、実時間（重複なし）で合計する
  - 例: 15:00〜16:30 と 16:00〜17:00 → 15:00〜17:00 = 2時間

## 集中できる時間帯の計算ルール

業務時間を **9:00〜18:00** と想定し、その範囲内でミーティングが入っていない連続した時間を探す。

- **確定ミーティングのみ**を使って空き時間を計算する（未回答イベントは除く）
- 30分未満の空き時間は除外する（短すぎて集中作業に向かないため）
- 業務時間外のミーティングは合計時間に含めるが、空き時間計算では9:00〜18:00の範囲のみを対象とする
- 空き時間がない場合は「集中できる時間帯はありません」と表示する
